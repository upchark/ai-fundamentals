## 4. Memory — Setting Custom Context

### Overview

Claude's **Memory / Custom Context** feature (accessed via Profile Settings) allows you to store persistent information about yourself, your business, and your preferences. Once set, Claude applies this context to every conversation automatically — eliminating the need to re-explain who you are or what you need each time.

### Key Concepts

**What is Memory in Claude?**  
Memory in Claude refers to **persistent user-level instructions** stored in your profile. This is not session-based memory (which only lasts the length of one chat) but permanent context that Claude carries into every new conversation.

**How to Set Custom Context (Memory)**
1. Go to Claude's **Profile / Settings**
2. Navigate to the **Memory** or **Custom Instructions** section
3. Write details about yourself: your name, your company, your role, your tone preferences, your target audience, and your common tasks
4. Claude will automatically use this information in all future chats

**What to Include in Your Memory/Context**
- Your name and role (e.g., "I am a Digital Marketing Manager at WsCube Tech")
- Your company name and what it does
- Your target audience
- Tone preferences (formal, casual, witty, etc.)
- Your most common tasks (e.g., "I mostly create promotional banners and social media content")
- Brand guidelines (colors, voice, keywords to avoid/use)
- Languages you prefer responses in

**Personalization Examples from the Video**
The instructor set up context for WsCube Tech including:
- Company name and its work (content and marketing for WsCube Tech and Ateck brand)
- Types of work: creating promotional materials, banners, social content
- Top-of-mind priorities and brand focus areas

With this context set, when you ask Claude "Write an Instagram caption," it automatically produces output aligned to WsCube Tech's brand, audience, and tone — without you needing to explain anything.

### Examples

**Without Memory:**  
User: *"Write a promotional post for our new course."*  
Claude: Asks — "What company? What course? What tone? Who is the audience?" → Multiple follow-up rounds needed.

**With Memory:**  
User: *"Write a promotional post for our new course."*

### Common Doubts

**Q: How is Memory different from just starting a chat with context?**  
A: When you paste context at the start of a chat, it only lasts that session. Memory/Profile settings persist across ALL chats, so you never have to repeat yourself. It's like onboarding Claude once so it always knows your context.

**Q: Can I have different memory for different projects?**  
A: For project-level context, use the **Projects** feature (covered in Section 9) which allows separate instructions per project. Profile memory is global; Projects memory is per-project.

**Q: What happens if I don't set up memory?**  
A: Claude treats you as a generic user. You'll get generic responses that require more editing and re-prompting to match your actual needs.

### Interview Questions

## 5. Claude Models — Haiku, Sonnet, Opus

### Overview

Claude is not a single model — it's a **family of models** with different capability levels and use cases. Choosing the right model for your task is crucial for both output quality and efficient token usage (cost management).

### Key Concepts

**The Three Model Tiers**

`📌 MEMORIZE` — Claude's model family as of 2026:

**1. Claude Haiku (Fastest, Lightest)**
- Fastest and most economical model
- Best for: Simple, quick tasks; casual conversations; basic Q&A; rapid iteration
- Trade-off: Less nuanced, less creative, shorter reasoning chains
- Use when: You need a quick answer or first draft and aren't concerned about maximum quality
- Token usage: Lowest → cheapest

**2. Claude Sonnet (Balanced — Most Commonly Used)**
- The "sweet spot" model: good balance of speed, quality, and cost
- Best for: Most professional tasks — writing, research, content creation, code generation
- Versions covered: Sonnet 4.5, **Sonnet 4.6** (latest recommended as of the video)
- Use when: You need reliable, high-quality output for real work
- Token usage: Moderate
- `📌 NOTE:` The instructor uses Sonnet 4.6 as the default model throughout this course

**3. Claude Opus (Most Powerful, Most Resource-Intensive)**
- The most intelligent and capable model
- Best for: Complex reasoning, intricate tasks, deep analysis, advanced code, long-form content requiring maximum quality
- Trade-off: Slower and uses more tokens (costs more)
- Use when: Quality matters more than speed; tasks that require deep reasoning
- Versions: Opus 3, Opus 4.5, Opus 4.6
- Token usage: Highest

**Practical Comparison Exercise from the Video**

The instructor demonstrated generating an Instagram Reel script using:
1. **Haiku** → Decent output, basic structure, quick
2. **Sonnet 4.5** → Noticeably improved, better narrative, more creative hooks
3. **Sonnet 4.6** → Further improvement, more professional structure
4. **Opus** → Highest quality output — richer scripts, better hooks, more strategic angles

The demonstration proved: the more capable the model, the more token usage it consumes, but also the better the output quality.

**Token Usage & Cost Management**
- Each model consumes a different number of tokens per response
- Better prompts = more concise responses = fewer tokens used
- Tip: Use Haiku for testing/iteration, Sonnet for daily work, Opus for final/complex outputs

### Terminology

| Term | Definition |
|---|---|
| **Model** | A specific trained version of Claude's LLM (e.g., Sonnet 4.6) |
| **Token** | Unit of text measurement; roughly ¾ of a word |
| **Token Usage** | How many tokens a model consumes per interaction |
| **Haiku** | Claude's fastest, lightest model |
| **Sonnet** | Claude's mid-tier, most-used model |
| **Opus** | Claude's most powerful model |

### Common Doubts

**Q: Which model should I use by default?**  
A: **Sonnet** (specifically Sonnet 4.6) is the recommended default for most professional tasks. It gives great quality without excessive token consumption. Switch to Opus when the task demands maximum intelligence; use Haiku for quick, low-stakes tasks.

**Q: Does model selection affect cost?**  
A: Yes, on the API and paid plans. Opus uses the most tokens and is the most expensive per output. If you're building an application on Claude's API, choosing the right model significantly impacts your cost.

**Q: Is Opus always better?**  
A: For most day-to-day tasks, Sonnet is sufficient and faster. Opus's advantage shows mainly in highly complex, multi-step tasks requiring deep reasoning. Using Opus for simple tasks is inefficient.

### Interview Questions

**Q: Explain the Claude model family and when you would use each.**  
A: Claude has three tiers — Haiku (fast, economical, for simple tasks), Sonnet (balanced, best for most professional work), and Opus (most powerful, for complex reasoning and high-quality outputs). Model choice should be driven by task complexity and cost considerations.

**Q: What is a token and why does model selection affect token usage?**  
A: Tokens are the units an LLM uses to process and generate text. Different models have different architectures and verbosity levels — more capable models like Opus tend to produce longer, richer outputs and also have higher per-token processing costs.

### Industry Usage

- **API developers** carefully select models based on cost/quality tradeoffs in their applications
- **Enterprise users** often standardize on Sonnet for daily tasks and reserve Opus for high-stakes outputs
- **Startups** often use Haiku for high-volume, low-complexity automation tasks to minimize API costs

### Revision Notes

- Haiku = Fast + Cheap + Simple tasks
- Sonnet = Balanced + Best for daily work + Default recommendation
- Opus = Most powerful + Complex tasks + Higher cost
- Better prompts reduce token usage regardless of model

## 6. Managing Chats and Tasks

### Overview

Effective chat management in Claude means knowing how to organize conversations, switch models mid-session, use different conversation modes, and structure tasks for maximum output quality.

### Key Concepts

**Creating New Chats**
- Each new chat starts fresh (unless you're inside a Project with persistent context)
- Use `+ New Chat` to start a clean session
- Best practice: Use separate chats for separate tasks to avoid context mixing

**Model Selection Per Chat**
- You can select which Claude model to use for each new chat
- This allows you to run the same prompt on different models to compare outputs (as demonstrated in the video — Haiku → Sonnet 4.5 → Sonnet 4.6 → Opus)

**Chat Modes**
- **Standard Chat**: Regular back-and-forth conversation
- **Task Mode**: For longer, multi-step tasks where Claude needs to take sequential actions
- **Projects**: For persistent, ongoing work with memory and file context (see Section 9)

**Comparing Model Outputs (Practical Technique)**
The video demonstrates a powerful technique: run the same prompt across multiple models to see the difference in quality. Start with Haiku, then upgrade to Sonnet, then Opus. This helps you:
1. Understand which model is "good enough" for your specific task
2. Save tokens by not defaulting to Opus for everything
3. Build intuition for model capabilities

**Task: Instagram Reel Script (From the Video)**  
Prompt used: *"Write an Instagram Reel script on the topic of [Digital Marketing topic]"*  
- Haiku: Basic structure, functional script, standard hooks
- Sonnet: Better narrative flow, improved engagement hooks, more strategic
- Opus: Richest content, most compelling hooks, professional-grade output

### Common Doubts

**Q: Should I always use the most powerful model for best results?**  
A: No. For most tasks, Sonnet 4.6 is excellent and more efficient. Use Opus selectively for tasks requiring complex multi-step reasoning or where output quality is critical.

**Q: How do I organize multiple ongoing projects without losing context?**  
A: Use the **Projects** feature (Section 9). Projects maintain separate context, instructions, and file uploads per project, so each project has its own memory.

### Revision Notes

## 7. Skills

### Overview

**Skills** in Claude are reusable custom instruction sets (essentially saved system prompts) that tell Claude *how* to approach specific types of tasks. Think of them as pre-trained workflows or expertise modules you build and store inside Claude.

### Key Concepts

**What Are Skills?**  
Skills are saved prompt frameworks stored in Claude's interface under **Customize → Skills**. When activated, a Skill tells Claude to follow specific rules, adopt a particular persona, use specific formats, or apply specialized knowledge for a given context.

`📌 MEMORIZE` — Skills = Reusable instruction templates stored in Claude for specific task types.

**Why Skills Matter**
- Without a Skill: Every time you want Claude to write social media content, you have to re-explain your content style, format, tone, platform rules, and requirements
- With a Skill: You activate your "Social Media Skill" and Claude instantly knows all of that — every time, automatically

**Built-in Skills (Pre-loaded by Claude)**
Claude comes with several built-in Skills including:
- **Web Artifact Builder** — Helps create HTML/CSS/JS mini-apps and widgets
- **Canvas Design** — For design-related outputs
- **MCP Builder** — For building MCP server configurations
- **Doc Skills** — For structured document creation
- **Theme Skills** — For applying visual/design themes
- **Brand Guidelines Skills** — For maintaining brand voice and rules

**Creating Your Own Custom Skills**  
Two methods:

**Method 1 — Using ChatGPT to generate a Skill**
1. Open ChatGPT
2. Ask: *"I want to create a Social Media Skill for Claude. Give me the skill instructions."*
3. ChatGPT will generate a structured skill template with:
   - Role-based content generation examples
   - Hook engineering instructions
   - Platform-specific guidelines
   - Tone and style rules
4. Copy the output into a Google Doc
5. In Claude → Customize → Skills → `+` → Upload Skill → Import the content

**Method 2 — Using Claude itself to generate a Skill**
1. In Claude, go to Customize → Skills → `+` → "Build with Claude"
2. Type: *"I want a Social Media Skill. Build it for me."*
3. Claude generates a complete Skill document
4. Review, edit, and save it
5. The Skill is now available in your library

**How to Test a Skill**
- Activate the Skill
- Use the slash command `/` to trigger it explicitly
- Run a test prompt relevant to the Skill's purpose
- Check if output quality improves vs. not using the Skill

**Skill Triggers (Slash Commands)**  
Each Skill can be assigned a `/command` (e.g., `/social`, `/content`, `/code-review`). Type the command in any chat to instantly invoke that Skill's instructions.

### Examples

**Q: Can I have multiple Skills active at once?**  
A: Yes. You can have a Social Media Skill and a Brand Guidelines Skill both active, and Claude will apply both when relevant.

**Q: Do I need to manually activate a Skill each time?**  
A: You can set default Skills that auto-apply, or manually trigger them with slash commands. For task-specific work, using `/command` trigger is recommended for precision.

### Interview Questions

**Q: What are Claude Skills and how do they improve output quality?**  
A: Skills are saved instruction frameworks in Claude that define how it should approach specific task types — tone, format, rules, persona, and methodology. Instead of re-prompting these rules every time, Skills make them persistent and reusable, dramatically improving consistency and quality of specialized outputs.

## 8. Connectors

### Overview

**Connectors** are Claude's integration layer — they allow Claude to directly access and interact with external tools, services, and data sources. With Connectors, Claude can read your Gmail, check your Notion workspace, interact with Stripe data, and much more — all without you leaving the Claude interface.

### Key Concepts

**What Are Connectors?**  
Connectors are third-party service integrations that give Claude real-time access to external platforms. They are an implementation of the **MCP (Model Context Protocol)** standard, which lets Claude communicate with external services.

`📌 MEMORIZE` — Connectors = Claude's bridge to the outside world; they give Claude real-time access to your tools and data.

**Available Connectors (as of 2026)**
From the video, available connectors include:
- **Gmail** — Read, summarize, and manage emails
- **Notion** — Access and create notes/pages
- **Stripe** — Access payment and customer data
- **Perplexity** — Real-time web search integration
- **n8n** — Workflow automation platform
- **And more...** (Browse via Connectors marketplace in Claude settings)

**How to Set Up a Connector (Gmail Example from Video)**
1. Go to Claude → Settings → Connectors (or Customize → Connectors)
2. Click **Browse Connectors**
3. Find **Gmail** and click to add
4. Toggle it **ON** for your account
5. Authorize Claude to access your Gmail (OAuth flow)
6. Now when you ask *"What is my latest email?"*, Claude reads your Gmail and responds

**Live Demo from Video:**  
The instructor added the Gmail connector and asked: *"What is my latest email? Let me know."*  
Claude responded: *"Latest email is a Security Alert from Google, received April 16th."* — Real-time data, no manual checking needed.

**Connectors vs. MCP Servers**
- Connectors = Pre-built, official integrations you can add with one click
- MCP Servers = Custom integrations you configure yourself for services not in the official list
- Both use the MCP protocol under the hood

**Important Limitations**
- You can connect MCP servers for external tools but Claude itself does not become a separate connector/plugin for other services
- Connector availability may expand over time as Anthropic adds more official integrations

### Examples

## 9. Projects

### Overview

**Projects** in Claude are persistent, organized workspaces for ongoing multi-session work. Unlike regular chats (which are isolated), Projects maintain context, instructions, and uploaded files across all conversations within that project.

### Key Concepts

**What Is a Project?**  
A Project is a dedicated container in Claude that holds:
- **Custom instructions** (how Claude should behave in this project's context)
- **Uploaded files** (documents, PDFs, data files that Claude can reference)
- **All chat history** within the project (persistent across sessions)

`📌 MEMORIZE` — Project = persistent workspace with its own instructions, files, and memory.

**Projects vs. Regular Chats**
| Feature | Regular Chat | Project |
|---|---|---|
| Memory across sessions | ❌ No | ✅ Yes |
| Custom instructions | ❌ No (unless in Profile) | ✅ Per-project |
| File uploads | ✅ Session only | ✅ Persistent |
| Multiple conversations | Separate | ✅ Unified |
| Best for | One-off tasks | Ongoing work |

**How to Create a Project**
1. In Claude's left sidebar, find **Projects** section (under Chats)
2. Click **+ New Project**
3. Give it a name (e.g., "WsCube Tech Social Media")
4. Add **Custom Instructions** — tell Claude who this project is for, the tone, format, goals
5. Upload relevant **Files** — brand guidelines, style documents, content calendars
6. Start chatting — Claude will always respond within this project's context

**Live Demo from Video:**  
The instructor had a project with:
- Instructions defining the brand's voice, audience, and content type
- An uploaded document with brand information
- When asking "Write a profile for us," Claude immediately knew the company, audience, and tone — because all context was stored in the project

**Project-Level Instructions vs. Profile Memory**
- **Profile Memory** = global context (applies everywhere)
- **Project Instructions** = specific context for this project only (can override or extend Profile Memory)
- Best practice: Keep general preferences in Profile; keep client/project-specific info in Projects

**Default Project**
Claude includes a **"How to Use Claude"** default project that serves as a tutorial — useful for beginners exploring Claude's capabilities.

### Examples

**Example Project Setup — "Client A Social Media"**
- Instructions: "You are a social media manager for [Client A]. Tone: Professional but approachable. Audience: B2B SaaS founders. Post formats: LinkedIn articles and short-form tweets."
- Files uploaded: Brand guide PDF, past campaign performance sheet, competitor analysis
- Result: Every new chat inside this project instantly produces client-ready content

### Common Doubts

**Q: How many projects can I have?**  
A: This depends on your plan, but in general, you can have multiple active projects. Organize by client, subject, or workflow.

**Q: Can I share a project with a team member?**  
A: As of this video, Projects are primarily individual. Check Anthropic's latest documentation for team collaboration features.

**Q: What's the difference between a Project and a Skill?**  
A: A **Skill** defines *how* Claude does a specific task type (methodology). A **Project** defines *what* Claude knows about a specific ongoing context (environment, client, goals). You can use Skills *inside* a Project.

### Interview Questions

**Q: How do Projects in Claude improve productivity for ongoing work?**  
A: Projects provide persistent context, instructions, and file access across all sessions within a workspace. This eliminates the repeated re-establishment of context for ongoing work (e.g., client projects), ensures consistency in tone and approach, and centralizes all related conversations in one organized workspace.

### Industry Usage

- **Freelancers**: Separate project per client for always-on-brand output
- **Dev teams**: Project for each codebase with architecture docs and coding standards uploaded
- **Marketing agencies**: Project per campaign with brief, audience data, and brand guidelines
- **Writers**: Project per book/series with character profiles, plot outlines, and style rules

### Revision Notes

- Project = persistent workspace with memory + custom instructions + files
- Separate projects for separate clients/topics
- Profile Memory = global; Project Instructions = local/specific
- Files uploaded to a Project are always available in that context
- Different from Skills (Skills = how; Projects = what context)

**Artifacts** in Claude are the visual, interactive outputs Claude creates — mini-apps, websites, dashboards, documents, and other structured content that can be previewed, edited, and published directly within Claude's interface.

**What Are Artifacts?**  
When you ask Claude to create something visual or functional — a website mockup, a task manager app, a data table, a simple game, or an interactive document — Claude generates it as an **Artifact**. This appears as a live preview panel alongside the chat.

- **Task Manager Artifacts** — Interactive task-tracking interfaces
